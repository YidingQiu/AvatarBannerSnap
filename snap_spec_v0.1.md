# LinkedInSnap — Product Specification v0.2

**A seamless LinkedIn profile photo + banner alignment tool**
*Future-extensible to any social platform*

---

## 1. Overview

The user uploads a single photo, positions it within a visual editor that simultaneously shows the LinkedIn avatar circle and banner rectangle as overlay guides, applies transforms (drag / zoom / rotate / flip), and exports two ready-to-upload images. The result: avatar and banner appear to be one continuous image on the profile page.

**Phase 1 deliverables**

| Artifact | Notes |
|---|---|
| Standalone Web App | Zero backend, pure client-side, deployable to Vercel / Cloudflare Pages |
| Chrome Extension | Deferred to Phase 2 |

Both share an identical React editor component (`packages/editor`). The difference is only the host environment.

## 2. Platform Calibration System

> **This is the foundation of the entire tool.** Every platform (LinkedIn, Twitter/X, GitHub, etc.) has its own banner dimensions and avatar placement. Getting these numbers wrong by even a few pixels breaks the illusion.

### 2.1 The Calibration Test Image

The test image uses a **2D position-encoding gradient**:

```
R channel = round(x / 3000 * 255)   — encodes horizontal position
G channel = round(y / 2000 * 255)   — encodes vertical position
B channel = 80                       — constant marker to isolate our pixels
```

After uploading to LinkedIn and taking a screenshot, sampling any pixel with B≈80
directly reveals the original (x, y) coordinates it came from.
No manual pixel-counting or circle detection needed.

Overlay (visual only, does not affect the gradient):
- Thin black grid lines every 100px
- Coordinate labels along the top and left edges only
- Center crosshair at (1500, 1000) — visual reference for avatar crop center

Generated by `apps/web/src/dev/CalibrationPage.tsx` (dev route `/dev/calibration`).
Download as PNG (lossless) before uploading to LinkedIn.
When LinkedIn prompts to reposition the profile photo, **accept the default** — do not drag.

### 2.2 Calibration Procedure (run once per platform)

1. Open `/dev/calibration` in the dev server, click **Generate** then **Download PNG**.
2. Upload the PNG to LinkedIn as both the **banner** and the **profile photo**.
   When prompted to reposition the profile photo, accept the default crop without dragging.
3. Take a screenshot of the profile page at the following viewport widths:
   - 1280 px
   - 1440 px
   - 1920 px
4. Run the detection script:
   ```
   pip install -r tools/requirements.txt
   python tools/detect_linkedin_calibration.py s1280.png s1440.png s1920.png
   ```
5. The script isolates B≈80 pixels, finds the banner rectangle and avatar circle
   from the gradient mask, and prints `cx_ratio`, `cy_ratio`, `d_ratio`.
   It also saves `*_annotated.png` files for visual verification.
6. If ratios are **stable across all viewports** (std < 0.01) — lock them in.
   If they **vary** — DOM-based real-time detection is needed (post-Phase 1 task).
7. Paste the printed constants into `packages/editor/src/platforms/linkedin.ts`
   and set `calibratedAt`.

## 3. Core Algorithm

The editor works in **canvas space**, with origin at the banner top-left corner.
The coordinate system extends downward to accommodate the avatar circle below.

```
Canvas coordinate space:

(0, 0) ──────────────────────────── (BannerW, 0)
  |                                       |
  |            BANNER REGION              |
  |                                       |
(0, BannerH) ─────────────── (BannerW, BannerH)
  |
  |  (avatar circle center is below banner bottom)
  o  <- (avatar_cx, avatar_cy)  where cy > BannerH
```

### 3.1 Photo Transform State

```typescript
interface PhotoTransform {
  translateX: number;  // offset from banner centre, display pixels
  translateY: number;
  scale: number;       // fabric scaleX/Y (photo px * scale = display px)
  rotation: number;    // degrees, -180 to +180
  flipX: boolean;
  flipY: boolean;
}
```

All transform operations are non-destructive and reversible until the user clicks Export.

Display coordinates are converted to export coordinates by multiplying by
`exportFactor = banner.exportW / DISPLAY_W` (currently `1584 / 880`).

### 3.2 Export Pipeline

```
1. Compute exportFactor = banner.exportW / DISPLAY_W

2. Render photo onto offscreen canvas (exportW wide, tall enough for avatar crop)
   applying PhotoTransform scaled by exportFactor

3. Crop banner region:
   rect(0, 0, banner.exportW, banner.exportH)
   -> save as linkedinsnap_banner.jpg @ 95% quality

4. Compute avatar bounding box:
   r  = (avatar.d_ratio  * banner.exportW) / 2
   cx = avatar.cx_ratio  * banner.exportW
   cy = avatar.cy_ratio  * banner.exportH
   rect(cx - r, cy - r, 2r, 2r)
   -> resize to avatar.exportSize x avatar.exportSize
   -> save as linkedinsnap_avatar.jpg @ 95% quality

5. Trigger download for both files (150 ms apart to avoid browser blocking).
```

## 5. UI / UX Specification

### Controls

| Control | Interaction |
|---|---|
| Move photo | Click + drag anywhere on canvas |
| Zoom | Scroll wheel |
| Rotate | Slider, -180 to +180 degrees |
| Flip horizontal | Toggle button |
| Flip vertical | Toggle button |
| Reset | Returns all transforms to identity |
| Export | Generates both files, triggers download |

### Overlay Layer (implemented via fabric.js non-interactive objects)

- Banner area: dashed rectangle, 2px white stroke, drop-shadow
- Avatar area: dashed circle, 2px white stroke, drop-shadow
- Overlays are `selectable: false, evented: false` — clicks pass through to photo

### Preview Panel

- Banner preview: 280px wide canvas, updates live via `requestAnimationFrame`
- Avatar preview: 80px circle canvas, clipped to circle shape
- Both re-render on every `PhotoTransform` change

### i18n

Language toggle (EN / Chinese) in header, persistent via `localStorage`.
Keys: `upload_hint`, `banner_label`, `avatar_label`, `rotation`, `flip_h`, `flip_v`,
`reset`, `download_btn`, `download_tip`, `preview_title`, `uncalibrated`.

### Key implementation decisions

| Decision | Choice | Rationale |
|---|---|---|
| Canvas interaction | fabric.js v6 | Drag + z-order management saves substantial custom code |
| Calibration method | 2D gradient (R=x, G=y, B=80) | Sample any pixel to read original coordinates — no fiducials or circle detection needed |
| Detection script | Python + OpenCV | Local run, no browser dependency for CV work |
| Monorepo tool | pnpm workspaces | Fast installs, symlinked workspace packages |
| Display width | 880px | Fits 1280px viewport with padding; scale factor = 1584/880 |
| Chrome Extension | Deferred to Phase 2 | Web app ships first |

### Platform Template

```typescript
export interface PlatformConfig {
  id: string;
  name: string;
  banner: { exportW: number; exportH: number; };

```

Adding a new platform = create one file, run calibration, fill in ratios.

- No photo data ever leaves the device
- Deploy to Vercel or Cloudflare Pages
